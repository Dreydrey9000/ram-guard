Gotchas

  • SAKANA CRITICALS (native engine, NOT fixed): C1 kill() can SIGTERM a recycled pid and hit the wrong process (5s scan-to-click window); C2 'quit app' only kills the biggest helper pid, so Chrome survives while the UI says it freed GBs; H3 readRam reports a FALSE 100% used when vm_stat returns empty (e.g. a hung call), which can trigger panic-quits.
  • The native app has the RAM core ONLY — disk views (junk/large/apps/login/storage) are NOT ported.
  • Native app is ad-hoc signed, local only, not notarized.
  • I could not screenshot Drey's screen (no Screen Recording permission); the native window UI is verified by clean launch only.
  • Build: swiftc -parse-as-library -O native/RAMGuard.swift; icon via node native/render-icon.js (Playwright resolves via NODE_PATH=~/My Apps/Carousels/glm-ursula-build/node_modules) then sips/iconutil.
